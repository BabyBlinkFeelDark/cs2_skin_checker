# 📈 Skin Watcher

Локальная утилита на Python для автоматического мониторинга цен на скины Steam-инвентаря. Проект спроектирован как невидимый фоновый агент для Windows и Linux. Он сканирует инвентарь пользователя, собирает историю цен в локальную SQLite-базу и присылает нативные уведомления о резких скачках или падениях стоимости предметов.

## 🌟 Новые возможности (v1.1)
- **Обход блокировок Steam:** Цены загружаются через открытый дамп стороннего API (без rate-лимитов и таймаутов).
- **Внешний файл настроек (`settings.json`):** Пороги алертов и SteamID больше не захардкожены в коде.
- **Аналитика и Алерты:** SQL-оконные функции сравнивают цены и генерируют сигналы на рост/падение прямо из базы.
- **Поддержка Прокси:** Встроенная поддержка HTTP/SOCKS прокси на случай блокировки эндпоинта инвентаря.

## 🏗️ Архитектура и основные модули

### 1. `steam_api.py` (Сбор данных)
* **`fetch_inventory()`** — Запрашивает открытый инвентарь пользователя. Использует подмену заголовка `Referer` для избежания ошибок `400 Bad Request`. Поддерживает сессии, ретраи (повторные попытки при таймауте) и прокси.
* **`fetch_price()`** — Вместо прямого парсинга SteamMarket (который банит IP), скрипт скачивает единый открытый JSON-дамп цен со стороннего источника, кэширует его в памяти и мгновенно находит цены всех скинов из инвентаря.

### 2. `services.py` (Бизнес-логика)
* **`sync_inventory()`** — Актуализирует состав инвентаря в базе, удаляя проданные предметы.
* **`refresh_prices()`** — Записывает новые точки цен в таблицу `price_history`.
* **`check_price_alerts()`** — Сравнивает последнюю записанную цену с предыдущей. Если изменение превышает порог из настроек (например, падение > 30%), вызывает модуль отправки уведомлений.

### 3. `database.py` (Хранилище)
Использует SQLite3. Таблицы:
* `market_items` — справочник уникальных предметов (чтобы не дублировать названия).
* `inventory_assets` — конкретные экземпляры в инвентаре пользователя.
* `price_history` — временной ряд цен (Time-Series) с автоматическими метками `TIMESTAMP`.

### 4. `config.py` (Настройки и Логи)
* **Динамические пути:** Если скрипт скомпилирован в `.exe`, база данных `app.db`, логи и `settings.json` автоматически направляются в скрытую системную папку `C:\Users\Имя\AppData\Local\skin_watcher\`.
* **Авто-настройка:** При первом запуске скрипт спрашивает SteamID пользователя в консоли и генерирует `settings.json`.

### 5. `alerts_sender.py` (Оповещения)
Кроссплатформенный модуль. На Windows выводит красивые нативные Toast-плашки со звуком через библиотеку `win11toast`. На Linux использует системный `notify-send`.

## ⚙️ Настройка (`settings.json`)

После первого запуска создается файл конфигурации со следующими параметрами:
```json
{
    "steam_id_64": "76561198000000000",
    "drop_threshold_percent": 30.0,
    "rise_threshold_percent": 25.0,
    "check_interval_hours": 1
}
```
Вы можете редактировать этот файл в блокноте в любое время, чтобы изменить пороги срабатывания алертов.

## 🚀 Запуск из исходников (Linux / Windows)
Клонируйте репозиторий.

Создайте виртуальное окружение: `python -m venv venv`

Установите зависимости: `pip install -r requirements.txt` (Для Linux временно уберите win11toast из файла).

Запустите скрипт: `python src/main.py`. При первом запуске введите свой открытый SteamID64.

## 📦 Сборка в автономный .exe для Windows
Утилиту можно скомпилировать в единый фоновый исполняемый файл с помощью PyInstaller.

Перенесите исходники на Windows.

Установите Python, выполните `pip install -r requirements.txt` и `pip install pyinstaller`.

#### Соберите бинарник командой:

```text
pyinstaller --noconsole --onefile src/main.py --name skin_watcher
```

В папке dist/ появится skin_watcher.exe. Его можно поместить в папку "Автозагрузка" (shell:startup), и он будет работать абсолютно незаметно. Все настройки и база будут храниться в вашей папке AppData.
