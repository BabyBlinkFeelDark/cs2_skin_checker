# 📈 Skin Watcher

Локальная утилита на Python для автоматического мониторинга цен на скины Steam-инвентаря. Проект спроектирован как фоновый агент (Windows/Linux), который сканирует инвентарь пользователя, собирает историю цен в локальную SQLite-базу и присылает нативные уведомления об обновлениях.

## 🏗️ Архитектура и основные модули

Проект разделен на логические слои: работа с API, база данных, бизнес-логика (сервисы) и планировщик.

### 1. `steam_api.py` (Адаптер данных)
Отвечает за сетевое взаимодействие с неофициальным API торговой площадки и инвентаря Steam Community.
* **`fetch_inventory()`**
  * **Описание:** Запрашивает открытый инвентарь пользователя по SteamID64.
  * **Особенности:** Steam API возвращает список `assets` (идентификаторы) и `descriptions` (картинки, названия). Этот метод автоматически "склеивает" их по `classid`, фильтрует только продаваемые предметы (marketable) и возвращает чистый список словарей: `asset_id`, `market_hash_name` и `amount`. Включает подмену заголовка `Referer`, чтобы избегать ошибок 400 Bad Request.
* **`fetch_price(market_hash_name)`**
  * **Описание:** Получает текущую минимальную цену (lowest_price) и объем торгов (volume) для конкретного предмета на Торговой площадке Steam.
  * **Особенности:** Кодирует спецсимволы в названиях скинов (`urllib.parse.quote`). Возвращает цену в очищенном виде (float) в долларах США (currency=1).

### 2. `database.py` (Хранилище данных)
Управляет локальной SQLite-базой данных с использованием контекстных менеджеров для безопасных транзакций.
* **`init_db()`**
  * **Описание:** Создает схему базы данных при первом запуске, если она еще не существует.
  * **Структура таблиц:**
    * `market_items` — справочник уникальных предметов (уникальность по `app_id` + `market_hash_name`).
    * `inventory_assets` — конкретные экземпляры предметов в инвентаре пользователя. Привязаны к `market_items` через Foreign Key.
    * `price_history` — временной ряд цен (Time-Series) для построения графиков и аналитики.

### 3. `services.py` (Бизнес-логика)
Связующее звено между Steam API и базой данных.
* **`sync_inventory()`**
  * **Описание:** Выполняет полное обновление инвентаря в БД.
  * **Процесс:** Скачивает инвентарь -> добавляет новые уникальные имена в `market_items` -> удаляет старый слепок `inventory_assets` -> записывает актуальный состав инвентаря.
* **`refresh_prices()`**
  * **Описание:** Снимает слепок цен для текущего портфеля.
  * **Процесс:** Делает SQL-запрос `JOIN`, чтобы получить только те предметы, которые *сейчас* есть в инвентаре. Запрашивает цены и пишет их в `price_history`.
  * **Важно:** Содержит встроенную задержку (Rate Limit) через `time.sleep(3.0 - 6.0)` между запросами, чтобы избежать временной блокировки IP-адреса от Steam (ошибка 429).

### 4. `notifier.py` (Уведомления)
Кроссплатформенный модуль оповещений.
* **`send_toast(title, body)`**
  * **Описание:** Отправляет системные push-уведомления.
  * **Особенности:** Динамически проверяет операционную систему. На Windows использует библиотеку `win11toast` (нативные уведомления со звуком), на Linux использует системный `notify-send` через `subprocess`. В случае отсутствия библиотек тихо пишет сообщения (fallback) в локальный лог-файл.

### 5. `config.py` (Конфигурация и Логирование)
Отвечает за маршрутизацию путей и логи.
* **Определение среды:** Проверяет `sys.frozen`. Если скрипт скомпилирован в `.exe` (PyInstaller), пути БД и логов направляются в системную скрытую папку `AppData\Local\skin_watcher`. Если запущен из редактора — создает папку `data` в корне проекта.
* **`setup_logging()`**
  * Настраивает `RotatingFileHandler`, который автоматически ротирует логи (максимум 5 файлов по 2 МБ), предотвращая бесконечное разрастание файла на жестком диске.

## 🚀 Установка и Запуск (Для разработки)

1. Клонируйте репозиторий.
2. Создайте виртуальное окружение: `python -m venv venv`
3. Установите зависимости: `pip install -r requirements.txt` (Для Linux временно уберите `win11toast` из файла).
4. В файле `src/main.py` впишите свой **SteamID64** в переменную `STEAM_ID`. Убедитесь, что ваш инвентарь Steam открыт в настройках приватности.
5. Запустите скрипт: `python src/main.py`

## 📦 Сборка в .exe для Windows

Проект готов к сборке в единый исполняемый файл с помощью PyInstaller.
