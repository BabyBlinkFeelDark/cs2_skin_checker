# 🔫 Skin Watcher

**Skin Watcher** — это самохостовый сервис для фонового мониторинга цен предметов Steam (CS2, Dota 2, Rust и др.) с отправкой алертов в Telegram и возможностью дальнейшего расширения веб-интерфейсом.

## Основные возможности

- Отслеживание цен по временным окнам: 24 часа, 7 дней и от точки входа (baseline).
- Фиксация точки входа для каждого инвентарного предмета и расчет профита относительно неё.
- Учет комиссии Steam (15 %) для оценки реального дохода на баланс.
- Антиспам по алертам (кулдауны на уровне предмет+окно+направление).
- Очереди отправки сообщений в Telegram с безопасным восстановлением после падений сети.
- Игнор «мусорных» предметов дешевле заданного порога, экономия лимитов API Steam.
- Базовая защита от «фейковых» пампов через проверку объема продаж.

## Архитектура

Проект реализован как модульный монолит:

- **Domain / Models** — dataclass-модели (инвентарь, цены, алерты, сообщения очереди).
- **Repositories** — слой доступа к SQLite (инвентарь, история цен, очереди, настройки).
- **UseCases** — сценарии: синхронизация инвентаря, обновление цен, анализ алертов.
- **Infrastructure** — клиенты Steam API, Telegram, системные уведомления (toast / notify-send).
- **EntryPoint** — `main.py`, который собирает зависимости, поднимает планировщик и оркестрирует пайплайн.

Хранилище — SQLite (`data/app.db`), все настройки вынесены в таблицу `app_settings`, что облегчает будущее подключение веб-интерфейса.

## Требования

- Python 3.10+
- Доступ к интернету для Steam и Telegram API
- Windows 10/11 (целевой рантайм для сборки в `.exe`, но разработка возможна и под Linux)

## Установка и запуск (режим разработки)

1. Клонировать репозиторий:

```bash
git clone https://github.com/yourusername/skin-watcher.git
cd skin-watcher
```

2. Установить зависимости (минимальный набор):

```bash
pip install -r requirements.txt
```

3. Первый запуск:

```bash
python src/main.py
```

При первом запуске будут созданы:

- каталог `data/`;
- база `app.db` с необходимыми таблицами;
- записи в `app_settings` с дефолтными значениями.

Далее нужно заполнить настройки (SteamID64, токены и chat_id бот(ов)) либо через существующий JSON (мигрируется автоматически), либо через будущий веб-/CLI-настройщик.

## Основной цикл работы

Периодически (по расписанию, интервал берется из `app_settings`):

1. **SyncInventoryUseCase** — подтягивает инвентарь из Steam, делает soft-sync с БД.
2. **UpdatePricesUseCase** — обновляет цены по предметам, игнорируя дешевые.
3. **AnalyzeAlertsUseCase** — анализирует динамику цен по окнам и создает алерты.
4. **AlertDispatcher** — склеивает сообщения в пачки и отправляет их в Telegram.

Все алерты и ошибки логируются в `data/app.log`.

## Сборка под Windows в один бинарник

Для сборки в `.exe` рекомендуется использовать PyInstaller с кастомным `.spec` файлом, в котором нужно:

- добавить в bundle каталог `data/` (или создать его при первом запуске); 
- добавить модули `templates/` и `static/` (когда появится веб-морда);
- указать `main.py` как точку входа.

Примерно:

```bash
pyinstaller --onefile --noconsole src/main.py -n SkinWatcher
```

(фактический командлайн и .spec нужно настраивать под конкретную структуру проекта и будущий веб-интерфейс).

## Планируемое развитие

- Локальный веб-интерфейс (FastAPI + Jinja2/HTMX) для настройки и визуализации.
- Вишлист для отслеживания целевых цен по предметам вне инвентаря.
- Детектор аномальных объемов (Volume Spikes) как отдельный сценарий анализа.
- Упрощенный мастер-настройки при первом запуске `.exe` на Windows.
